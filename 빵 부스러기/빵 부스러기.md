### 1. MySQL 아키텍처 관련 질문
- MySQL의 주요 구성 요소는 무엇이며, 각 역할은 무엇인가요?
- MySQL 엔진과 스토리지 엔진의 차이는 무엇인가요?
- 핸들러 API는 어떤 역할을 하나요?
- MySQL이 제공하는 클라이언트 접속 방식에는 어떤 것들이 있나요?


### 2. 쿼리 실행 과정 관련 질문
- 쿼리 실행 과정에서 가장 먼저 수행되는 단계는 무엇인가요?
- MySQL 8.0에서 쿼리 캐시가 제거된 이유는 무엇인가요?
- 쿼리 파서의 역할과 Parse Tree의 개념을 설명해주세요.
- 전처리기가 하는 역할은 무엇인가요?
- 옵티마이저는 어떤 방식으로 쿼리를 최적화하나요?
- 규칙 기반 최적화와 비용 기반 최적화의 차이는 무엇인가요?
- 실행 엔진은 어떤 역할을 담당하나요?
- MySQL에서 트랜잭션 격리 수준이 중요한 이유는 무엇인가요?


### 3. InnoDB 스토리지 엔진 관련 질문
- InnoDB의 주요 특징 3가지는 무엇인가요?
- 클러스터링 인덱스(Clustered Index)란 무엇이며, 어떤 장점과 단점이 있나요?
- PK(프라이머리 키) 기반으로 데이터가 저장된다는 것이 어떤 의미인가요?
- InnoDB에서 PK 순으로 데이터를 저장하면 어떤 장점이 있나요?
- InnoDB에서 PK를 통한 범위 검색이 빠른 이유는 무엇인가요?
- PK 변경이 쓰기 성능을 저하시킨다는 것은 어떤 의미인가요?
- MVCC(다중 버전 동시성 제어)란 무엇이며, 어떻게 동작하나요?
- 언두 로그와 리두 로그의 차이는 무엇인가요?
- 언두 로그가 트랜잭션 격리 수준을 보장하는 방식은 무엇인가요?
- 언두 로그가 롤백에 활용되는 방식은 어떻게 되나요?
